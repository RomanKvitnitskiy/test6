КОМУТАТОР 3 РІВНЯ

           Комутатор 3 рівня (також відомий як маршрутизатор) є мережевим пристроєм, який виконує ключову роль у мережевому інфраструктурі. Основною його функцією є маршрутизація даних між різними мережами на основі інформації з IP-протоколу.

Основні функції комутатора 3 рівня включають:

           1) Маршрутизація: Комутатор 3 рівня аналізує заголовки пакетів даних, що проходять через нього, і вирішує, куди направити ці пакети на основі інформації про мережевий шлях. Він використовує IP-адреси пакетів для визначення найкращого шляху до призначення.

           2) Маршрутизація зворотного зв'язку: Комутатор 3 рівня здатний відправляти відповіді на запити, отримані від пакетів даних, назад до початкового вузла. Це важливо для забезпечення двостороннього зв'язку в мережі.

           3) Фільтрація даних: Комутатор 3 рівня може виконувати фільтрацію даних на основі різних критеріїв, таких як IP-адреси призначення та джерела, номери портів та протоколи. Це дозволяє контролювати доступ до різних ресурсів мережі та забезпечити безпеку мережі.

           4) NAT (Network Address Translation): Комутатор 3 рівня може виконувати функцію перетворення IP-адрес мережі, що називається NAT. Вона дозволяє внутрішнім пристроям використовувати одну зовнішню IP-адресу для з'єднання з Інтернетом. NAT забезпечує захист внутрішньої мережі та економію IP-адрес.

           5) QoS (Quality of Service): Комутатор 3 рівня може використовувати технологію QoS для призначення пріоритетів пакетам даних в мережі. Це дозволяє пріоритезувати різні типи трафіку, такі як голосові або відеоданих, щоб забезпечити високу якість обслуговування для додатків, які вимагають низької затримки або високої пропускної здатності.

           6) Сегментування мережі: Комутатор 3 рівня дозволяє створювати різні сегменти мережі, розділяючи її на логічні частини. Це дозволяє керувати трафіком і підвищує безпеку, оскільки він обмежує зв'язок між сегментами мережі.

           7) Підтримка протоколів мережевої безпеки: Комутатор 3 рівня підтримує різні протоколи мережевої безпеки, такі як VPN (Virtual Private Network), фаерволи, IPSec (Internet Protocol Security) і т.д. Він забезпечує захист мережі від несанкціонованого доступу та інших загроз.

Ці функції дозволяють комутатору 3 рівня керувати рухом даних в мережі, забезпечувати надійність, безпеку та ефективне використання мережевих ресурсів.




ОСНОВНІ НАЛАШТУВАННЯ

  • Маршрутизація


               Основні команди налаштування маршрутизації на пристроях Cisco в залежності від версії операційної системи IOS можуть трохи відрізнятися. Ось кілька загальних команд для налаштування маршрутизації на пристроях Cisco:

               1) Налаштування IP-адреси і маршруту за замовчуванням:

    Router(config)# interface <інтерфейс>
    Router(config-if)# ip address <IP-адреса> <Маска підмережі>
    Router(config-if)# no shutdown
    Router(config)# ip default-gateway <IP-адреса>


               2) Додавання статичного маршруту:

    Router(config)# ip route <Мережа призначення> <Маска підмережі> <IP-адреса наступного хопу або інтерфейс>


               3) Включення протоколу маршрутизації OSPF (Open Shortest Path First):

    Router(config)# router ospf <Номер процесу OSPF>
    Router(config-router)# network <Мережа> <Маска підмережі> area <Номер області>


               4) Включення протоколу маршрутизації RIP (Routing Information Protocol):

    Router(config)# router rip
    Router(config)# network <Мережа>


               5) Налаштування маршрутизації між VLAN на комутаторі з підтримкою маршрутизації:

    Switch(config)# interface vlan <Номер VLAN>
    Switch(config-if)# ip address <IP-адреса> <Маска підмережі>


               6) Перегляд таблиці маршрутизації:

    Router# show ip route


               Це лише деякі базові команди для налаштування маршрутизації на пристроях Cisco. Уточнена синтаксис та доступні команди можуть залежати від версії IOS та конфігурації пристрою. Рекомендується звертатися до документації Cisco або офіційних ресурсів для отримання докладнішої інформації.
  • Маршрутизація зворотного зв'язку


               Маршрутизація зворотного зв'язку (reverse path forwarding, RPF) є важливою функцією для контролю надійності та безпеки мережі. Вона перевіряє, чи маршрутизований трафік приходить через той самий інтерфейс, з якого був отриманий початковий пакет. Налаштування маршрутизації зворотного зв'язку в Cisco може варіюватися залежно від конфігурації і версії IOS, але основні кроки включають наступне:

               1) Включити функцію маршрутизації зворотного зв'язку на інтерфейсах:

    Router(config)# interface <інтерфейс>
    Router(config-if)# ip verify unicast reverse-path


               2) Вибрати режим перевірки RPF. Існують два основних режими: строгий (strict) та небрежний (loose). Режим строгої перевірки (strict mode) вимагає, щоб трафік повертався через той самий інтерфейс, через який був отриманий початковий пакет. Режим небрежної перевірки (loose mode) дозволяє трафіку повертатися через будь-який доступний інтерфейс, якщо він має маршрут до джерела пакета.

    Router(config)# ip verify unicast source reachable-via <режим>


               3) Налаштувати додаткові параметри, які впливають на маршрутизацію зворотного зв'язку, такі як періодичність оновлення маршрутної таблиці та настройки логування:

    Router(config)# ip cef
    Router(config)# ip verify unicast source reachable-via all
    Router(config)# ip verify unicast source reachable-via rx allow-default
    Router(config)ip verify unicast source reachable-via rx maximum-log-disruptions 50


               4) Перевірте стан маршрутизації зворотного зв'язку:

    Router# show ip interface <інтерфейс>
    Router# show ip cef
    Router# show ip verify source reachable-via


               Зверніть увагу, що конфігурація маршрутизації зворотного зв'язку може бути складною, особливо в складних мережевих топологіях. Рекомендується ретельно вивчити документацію Cisco та консультуватися з фахівцями для впевненості у правильній настройці цієї функції.
  • Фільтрація даних


               Настроювання фільтрації даних на пристроях Cisco виконується за допомогою Access Control Lists (ACL). ACL дозволяють керувати рухом пакетів у мережі на основі різних критеріїв, таких як IP-адреси, порти і протоколи. Ось декілька команд для налаштування фільтрації даних на пристроях Cisco:

               1) Створення ACL:

    Router(config)# access-list <номер> {permit | deny} <протокол> <джерело> <призначення>


               Примітка: '<номер>' - це номер ACL, '<протокол>' - тип протоколу, '<джерело>' і '<призначення>' - діапазони IP-адрес або одиночні адреси.

               2) Використання ACL на інтерфейсах для фільтрації вхідного або вихідного трафіку:

    Router(config)# interface <інтерфейс>
    Router(config-if)# ip access-group <номер ACL>{in | out}


               Примітка: '<номер ACL>' - номер ACL, 'in' - для фільтрації вхідного трафіку, 'out' - для фільтрації вихідного трафіку.

               3) Перегляд інформації про ACL:

    Router# show access-lists


               4) Виконання додаткових дій з ACL, таких як редагування або видалення:

    Router(config)# no access-list <номер>


               Це лише основні команди для налаштування фільтрації даних на пристроях Cisco за допомогою ACL. ACL може мати різні варіації та параметри, що залежать від версії IOS та конфігурації пристрою. Рекомендується звернутися до документації Cisco або офіційних ресурсів для отримання докладнішої інформації про ACL та його налаштування.
  • NAT (Network Address Translation)


               Налаштування Network Address Translation (NAT) на пристроях Cisco можна виконати за допомогою різних команд, залежно від версії IOS та типу NAT, який ви хочете налаштувати. Ось декілька основних команд для налаштування NAT на пристроях Cisco:

               1) Створення статичного NAT:

    Router(config)# ip nat inside source static <локальна IP-адреса> <глобальна IP-адреса>


               Ця команда вказує, що локальна IP-адреса буде перекладена на глобальну IP-адресу.

               2)Створення динамічного NAT:

    Router(config)# ip nat inside source list <номер ACL> interface <інтерфейс> overload


               У цьому випадку, ACL (Access Control List) визначає діапазон локальних IP-адрес, які будуть перекладені, а 'interface' вказує на зовнішній інтерфейс маршрутизатора, через який здійснюватиметься NAT. 'overload' дозволяє використовувати портове перекладання (PAT), дозволяючи багатьом локальним IP-адресам використовувати одну глобальну IP-адресу.

               3) Налаштування пристрою як NAT-пристрою:

    Router(config)# interface <інтерфейс>
    Router(config- if)# ip nat inside


               Ця команда вказує, що вказаний інтерфейс є внутрішнім (inside) і NAT буде застосовуватися до пакетів, що проходять через цей інтерфейс.

               4) Встановлення зв'язку між внутрішніми та зовнішніми інтерфейсами NAT:

    Router(config)# interface <внутрішній інтерфейс>
    Router(config- if)# ip nat inside
    Router(config)# interface <зовнішній інтерфейс>
    Router(config- if)# ip nat outside


               Ці команди вказують, які інтерфейси є внутрішніми і зовнішніми для NAT.

               5) Перегляд інформації про NAT:

    Router# show ip nat translations
    Router# show ip nat statistics


               Ці команди дозволяють переглядати активні переклади NAT та статистику NAT на маршрутизаторі Cisco.

    Зверніть увагу, що налаштування NAT може варіюватися в залежності від конкретної версії IOS та моделі пристрою Cisco.
  • QoS (Quality of Service)


               Налаштування Quality of Service (QoS) на пристроях Cisco включає в себе використання різних команд для призначення пріоритетів трафіку, обмеження пропускної здатності та управління буферами. Ось декілька основних команд для налаштування QoS на пристроях Cisco:

               1) Визначення класів трафіку і пріоритетів:

    Router(config)# class-map <ім'я-класу>
    Router(config-cmap)# match <критерії>
    Router(config)# policy-map <ім'я-політики>
    Router(config-pmap)# class <ім'я-класу>
    Router(config-pmap-c)# <дія>


               Ці команди дозволяють визначити класи трафіку за допомогою 'class-map' і визначити правила для відповідного класу за допомогою 'policy-map'. Ви можете використовувати 'match' для встановлення критеріїв збігу для класу та 'дія' для визначення дії, яку необхідно застосувати до трафіку.

               2) Налаштування пріоритету через зворотну смугу:

    Router(config)# interface <інтерфейс>
    Router(config- if)# bandwidth <пропускна-здатність>


               Ця команда дозволяє налаштувати пропускну здатність для інтерфейсу. Вона використовується для розподілу пропускної здатності між різними класами трафіку на інтерфейсі.

               3) Встановлення обмежень швидкості:

    Router(config)# policy-map <ім'я-політики>
    Router(config-pmap)# class <ім'я-класу>
    Router(config-pmap-c)# policy <швидкість> <спосіб-поліціювання>


               Ці команди дозволяють встановити обмеження швидкості для певного класу трафіку. Ви можете вказати швидкість та спосіб поліціювання (наприклад, дозволяти, відкидати або маркувати пакети).

               4) Налаштування чергування через буфери:

    Router(config)# interface <інтерфейс>
    Router(config- if)# service-policy output <ім'я-політики>


               Ця команда дозволяє налаштувати чергування через буфери для вихідного трафіку на інтерфейсі. Ви можете вказати ім'я політики QoS, яку потрібно застосувати до вихідного трафіку.

               5) Перегляд статистики QoS:

    Router# show policy-map interface <інтерфейс>
    Router# show class-map


               Ці команди дозволяють переглядати статистику QoS, таку як статистику політик QoS на конкретному інтерфейсі або список класів трафіку.

               Зверніть увагу, що налаштування QoS може варіюватися залежно від конкретної версії IOS та моделі пристрою Cisco.
  • Сегментування мережі


               Для налаштування сегментування мережі на пристроях Cisco можна використовувати команди, пов'язані з VLAN (Virtual LAN) і маршрутизацією між VLAN. Ось кілька основних команд для налаштування сегментування мережі:

               1) Створення VLAN:

    Router(config)# vlan <номер-VLAN>


               Ця команда створює новий VLAN з вказаним номером. Ви можете створювати багато VLAN на комутаторі.

               2) Назначення порту до VLAN:

    Router(config)# interface <інтерфейс>
    Router(config- if)# switchport mode access
    Router(config- if)# switchport access vlan <номер-VLAN>


               Ці команди назначають порт комутатора до вказаного VLAN. Вказівка 'switchport mode access' встановлює режим порту як "access", а 'switchport access vlan <номер-VLAN>' призначає VLAN для порту.            3) Настроювання маршрутизації між VLAN:

    Router(config)# interface <інтерфейс>
    Router(config- if)# no shutdown
    Router(config- if)# description <опис-інтерфейсу>
    Router(config- if)# ip address <IP-адреса> <маска-підмережі>
    Router(config)# interface <інтерфейс>
    Router(config- if)# no shutdown
    Router(config- if)# description <опис-інтерфейсу>
    Router(config- if)# ip address <IP-адреса> <маска-підмережі>


               Ці команди виконують налаштування маршрутизатора для маршрутизації між VLAN. Вам потрібно налаштувати інтерфейси маршрутизатора з відповідними IP-адресами та масками підмережі для кожного VLAN.

               4) Налаштування VLAN Trunking Protocol (VTP):

    Switch(config)# vtp mode <режим>
    Switch(config)# vtp domain <ім'я-домену>
    Switch(config)# vtp password <пароль>


               Ці команди налаштовують VTP, що дозволяє автоматично розповсюджувати інформацію про VLAN між комутаторами. Ви можете встановити режим (server, client або transparent), вказати ім'я домену і встановити пароль для VTP.

               5) Перегляд інформації про VLAN:

    Switch# show vlan
    Switch# show interfaces <інтерфейс> switchport
    Switch# show vtp status


               Ці команди дозволяють переглядати інформацію про стан VLAN, налаштування портів комутатора та статус VTP відповідно.

               Зверніть увагу, що конкретні команди та синтаксис можуть змінюватися залежно від версії IOS та моделі комутатора Cisco.
  • Підтримка протоколів мережевої безпеки


               Налаштування підтримки протоколів мережевої безпеки на пристроях Cisco може включати в себе використання різних команд для налаштування захисту мережі, включаючи файрволи, віртуальні приватні мережі (VPN), аутентифікацію і авторизацію, а також контроль доступу. Ось кілька основних команд для налаштування підтримки протоколів мережевої безпеки на пристроях Cisco:

               1) Налаштування статичних або динамічних правил файрвола:

    Router(config)# access-list <номер-ACL> <тип> <джерело> <призначення>
    Router(config)# interface <інтерфейс>
    Router(config- if)# ip access-group <номер-ACL> <вхід-або-вихід>


               Ці команди дозволяють налаштувати правила файрвола на основі списків доступу (ACL). Ви можете визначити номер ACL, тип правила (дозволяюче або забороняюче), джерело та призначення трафіку. Потім ви можете застосувати ACL до вхідного або вихідного потоку на конкретному інтерфейсі.

               2) Налаштування віртуальної приватної мережі (VPN):

    Router(config)# crypto isakmp policy <політика-ISAKMP>
    Router(config-isakmp)# encryption <тип-шифрування>
    Router(config-isakmp)# hash <тип-хешування>
    Router(config-isakmp)# group <тип-групи>
    Router(config-isakmp)# lifetime <тривалість>
    Router(config)# crypto isakmp key <ключ> address <IP-адреса-віддаленого-пристрою>
    Router(config)# crypto ipsec transform-set <набір-трансформацій> <тип-шифрування>
    Router(config)# crypto map <назва-мапи> <пріоритет> ipsec-isakmp
    Router(config-crypto-map)# set peer <IP-адреса-віддаленого-пристрою>
    Router(config-crypto-map)# set transform-set <набір-трансформацій>
    Router(config)# interface <інтерфейс>
    Router(config- if)# crypto map <назва-мапи>


               Ці команди дозволяють налаштувати VPN на пристроях Cisco. Ви можете визначити політику ISAKMP (Internet Security Association and Key Management Protocol), тип шифрування, тип хешування, тип групи, тривалість ключа та ключ для шифрування трафіку. Також потрібно налаштувати набір трансформацій IPSec (Internet Protocol Security), мапу IPSec, вказати віддалені пристрої і застосувати мапу до відповідного інтерфейсу.

               3) Налаштування аутентифікації і авторизації:

    Router(config)# aaa new-model
    Router(config)# aaa authentication <тип-аутентифікації> <група> <метод>
    Router(config)# aaa authorization <тип-авторизації> <група> <метод>
    Router(config)# username <ім'я-користувача> password <пароль>


               Ці команди дозволяють налаштувати аутентифікацію і авторизацію на пристроях Cisco. Ви можете використовувати різні типи аутентифікації (наприклад, локальна аутентифікація, TACACS+, RADIUS) і визначити методи аутентифікації та авторизації для кожної групи користувачів. Також потрібно створити користувачів з іменами та паролями.

               4) Налаштування контролю доступу:

    Router(config)# access-list <номер-ACL> <тип> <джерело> <призначення>
    Router(config)# interface <інтерфейс>
    Router(config- if)# ip access-group <номер-ACL> <вхід-або-вихід>


               Ці команди дозволяють налаштувати списки доступу (ACL) для контролю доступу до мережі. Ви можете визначити номер ACL, тип правила (дозволяюче або забороняюче), джерело та призначення трафіку. Потім ви можете застосувати ACL до вхідного або вихідного потоку на конкретному інтерфейсі.

               Це лише кілька прикладів команд для налаштування підтримки протоколів мережевої безпеки на пристроях Cisco.
